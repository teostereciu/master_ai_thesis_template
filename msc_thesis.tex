        \chapter*{Abstract}
                \begin{adjustwidth}{4cm}{4cm}
                        AI agents are only as capable as the tools available to them, yet building those tools --- translating API documentation into callable, protocol-compliant interfaces --- remains a manual bottleneck that does not scale to the long tail of APIs agents need to be genuinely useful. This thesis asks whether large language models can replace that engineering step entirely.                                           We introduce an end-to-end benchmark in which a synthesis agent reads raw developer documentation for 14 REST APIs and produces a complete MCP server --- a runnable, deployable tool collection, not individual function stubs --- against which a separate evaluation agent attempts real tasks, with outcomes judged by a third model. Three synthesis conditions (with documentation, without documentation, and with deliberately renamed parameters conflicting with training-time knowledge) are evaluated across eight models spanning three capability tiers.
                        
                        The results challenge several assumptions. Synthesis succeeds broadly: eleven of fourteen APIs yield servers with sufficient tool coverage for their task sets, with a frontier-tier median agent pass rate of 83\%, and synthesized servers match or exceed human-authored references on GitHub and Zulip. Withholding documentation entirely does not hurt --- and sometimes helps --- because model weights already encode accurate implementations for well-known, mature APIs. For popular APIs such as GitHub and Stripe, parametric knowledge overrides in-context documentation even when that documentation explicitly conflicts with it, a phenomenon we term \emph{parametric gravity} and localise to early-to-mid transformer layers via logit-lens probing. Finally, how completely a synthesized server covers the required tool space strongly predicts downstream agent performance ($r = 0.937$, $p < 0.0001$, $n = 209$), providing a cheap pre-flight signal that can be assessed before any agent runs.
                        
                        Together, these findings suggest that automated tool synthesis is not a future prospect but a present capability --- one that could substantially reduce the engineering overhead standing between AI agents and the broader API ecosystem.
\end{adjustwidth}
